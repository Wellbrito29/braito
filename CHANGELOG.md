# Changelog

All notable changes to braito will be documented here.

## [Unreleased]

### Added
- **Quality harness (static mode)** — `tests/quality/qualityHarness.test.ts` runs `runGenerate` against a curated 6-file fixture repo (`tests/fixtures/quality-harness/`) spanning the common archetypes braito targets (HTTP controller, service with DB + HTTP client, DTO with zod, domain entity, React hook, Express middleware with env vars). Per-file expectations declare regex patterns that MUST match `purpose.observed`, `invariants.observed`, and `knownPitfalls.observed`, plus minimum item counts. An aggregate floor asserts that after enrichment the repo-wide average stays above known-good thresholds (purpose > 1.0, invariants > 0.5 per file). A failing expectation tells you exactly which static signal stopped firing — making prompt-tuning and enricher changes a safe operation instead of a guess. Runs in <600ms, no LLM needed, no API key required; guards against regressions in filename classification, import categorization, cross-signal inferences, AST extractors, and the aggregation wiring.
- **Claude CLI provider** — new `provider: 'claude-cli'` in `ai-notes.config.ts` spawns the local `claude` binary (`claude -p --output-format json`) for synthesis; authenticates via the user's existing Claude Code session, so no `ANTHROPIC_API_KEY` is required; `ClaudeCliProvider` in `src/core/llm/provider/claude-cli.ts` pipes the user prompt via stdin, passes the system prompt via `--system-prompt` (fully replaces the default system prompt to avoid user memory/preference leakage), parses the JSON result, and reuses the shared `withRetry` helper; surfaces `costUsd` and `durationMs` from the CLI JSON response; wired into `LLMProviderName`, `llmConfigSchema` enum, and `createProvider` factory; covered by new cases in `tests/llm/factory.test.ts` and `tests/config/configSchema.test.ts`
- **Governance divergence detection** — new `src/core/governance/detectDivergence.ts` cross-references governance docs against the actual codebase and graph; four detectors flag `missing_file` (docs reference a file that doesn't exist), `undeclared_domain` (file lives outside every domain declared by architecture docs), `forbidden_dependency` (graph edge violates a "`src/X` must not depend on `src/Y`" rule mined from doc prose), and `undocumented_hotspot` (≥5 reverse deps, no doc coverage); wired into `runGenerate` right after governance context load; per-file divergences injected into `knownPitfalls.observed` with `evidence.type: 'doc'`; all divergences persisted to `.ai-notes/divergences.json`; new `get_divergences` MCP tool with optional `severity`/`type` filters; 11 tests in `tests/governance/detectDivergence.test.ts`
- **Multi-repo MCP** — `mcp --roots "alias=/path,..."` registers multiple repositories with a single MCP server; each tool call accepts an optional `repo` argument routing to the matching registry entry; new `list_repos` tool enumerates registered repos; tool schemas now include a `repo` property; error responses returned when `repo` is omitted with multiple registered repos or when the alias is unknown; single-repo behavior (via `--root`) is unchanged — `repo` stays optional; `handleRequest` accepts either `(req, root, outputDir)` (legacy) or `(req, registry)` (new); helpers `parseRootsSpec` and `buildRegistry` exported; 14 tests in `tests/mcp/multiRepo.test.ts`
- **Graph UI — cycles, focus mode, and analysis panel** — `GET /api/graph/cycles` runs iterative Tarjan SCC on persisted `graph.json` (or `index.json` fallback) returning cycles and flat member set; `GET /api/graph/analysis?path=X` returns per-file graph position: in/out degree, transitive dependents/deps via BFS, neighbor domain distribution, cycle membership with sibling list, and `hotspot` flag (score ≥ 0.6 + ≥5 transitive consumers + no tests); Graph tab rewritten with Global/Focus toggle (focus renders ego-network around selected file), depth slider (1–3 hops), upstream edges blue / downstream red / cycle edges red with ringed members, domain-legend checkboxes to hide/show domains, and right-side analysis pane showing stats, hotspot warning, cycle membership, and signals; fallback fix: `/api/graph` (index-derived) had edge direction inverted — `dependents[]` means "files that import me", so edges now go `dep → entry`
- **Python/Go `exportDetails` and `signatures`** — both analyzers now extract rich export metadata: Python extracts function signatures with params/return types, class definitions with bases, and docstrings; Go extracts function signatures including methods with receivers, struct fields (up to 6), and interface methods; `signatures` field populated from extracted details for LLM prompt enrichment; Python respects `__all__` to filter exports; Python handles multiline `from X import (...)` blocks; Go captures exported methods with receivers; `ENSURES` and `ADR` keywords added to both comment extractors
- **BM25 search index** — `generate` now builds `.ai-notes/search-index.json` using MiniSearch (BM25 scoring); the MCP `search` tool uses ranked full-text search with fuzzy matching and prefix support instead of linear substring scanning; searches across all `observed[]`, `inferred[]`, and `evidence[].detail` fields; falls back to linear scan when index is not present; `searchMethod` field in response indicates which engine was used
- **Graph UI tab** — interactive dependency graph visualization in the `ui` command using D3.js force-directed layout; nodes colored by domain, sized by criticality score; directed edges show import relationships; click-to-detail opens file note; zoom/pan via mouse; drag to reposition nodes; hover highlights neighbors and dims unrelated nodes; tooltip shows path, score, and domain; min-score filter slider for large graphs; `GET /api/graph` endpoint serves `graph.json` or falls back to index dependents
- **Governance-aware analysis** — new `src/core/governance/` module detects project documentation artifacts (`Docs/`, `Workflows/`, `Quality/`, `Skills/`) and injects governance evidence into file notes; `detectGovernanceModel` scans well-known paths (NebulaSpecKit convention and common alternatives); `loadGovernanceContext` builds structured context with domain mappings, constraints, and decisions extracted from markdown headings; `buildBasicNote` produces `evidence.type: 'doc'` items linking files to their governing documents; `get_architecture_context` MCP tool enriched with governance summary; new `get_governance_context` MCP tool exposes detected docs, style, and domain mappings

- **`get_business_rules` MCP tool** — new tool that statically extracts business rules from a source file: numeric limits (`MAX_*`, `MIN_*`, `LIMIT_*`, `THRESHOLD_*` constants and numeric comparisons), permission guards (role checks, `ForbiddenException`/`UnauthorizedException` throws), schema validations (Zod/Joi/Yup chain methods and class-validator decorators), business constants (ALL_CAPS names and domain-term constants like PRICE, TAX, TIMEOUT), and conditional throws (domain invariant enforcement); returns `{ path, ruleCount, rules: BusinessRule[] }`; if an LLM is configured, appends an optional 2-3 sentence summary of the domain policies enforced; extractor implemented in `src/core/business/extractBusinessRules.ts` as a pure function

### Changed
- **Static-note enrichment v2 — filename, import, and cross-signal hints** — new `src/core/output/enrichStaticSignals.ts` merges three new layers into the observed[] arrays of every note: (1) *filename classification* — maps `*.controller.ts`, `*.service.ts`, `*.module.ts`, `*.guard.ts`, `*.filter.ts`, `*.middleware.ts`, `*.pipe.ts`, `*.interceptor.ts`, `*.dto.ts`, `*.entity.ts`, `*.enum.ts`, `*.use-case.ts`, `*.repository.ts`, `*.spec.ts`/`*.test.ts`, `useXxx.ts` React hooks, `index.ts` barrels, and `.tsx`/`.jsx` components to human-readable purpose labels; (2) *import categorization* — translates database ORMs (mongoose/typeorm/prisma/…), HTTP clients (axios/got/…), message queues (amqplib/kafkajs/bullmq/…), caches (redis/…), auth libs (jsonwebtoken/bcrypt/passport), logging libs (winston/pino/…), crypto, fs, child_process, observability backends (sentry/datadog/newrelic/otel), analytics (mixpanel/posthog/…), feature flags, and realtime (socket.io/ws) into invariants; (3) *cross-signal inferences* — "Consumed by N files but has no related tests", "low coverage + many consumers → regression blast radius", "env-var deps without tests", "volatile file (high churn × multiple authors)", and "side-effectful module imported by N consumers runs on every load". Hints are merged with case-insensitive dedup so existing entries aren't duplicated. Measured on the 74-file example NestJS repo: avg `purpose.observed` rose from 1.0 to 1.9, `invariants.observed` from ~0 to 0.2, `knownPitfalls.observed` from ~0 to 1.3 per static note. 34 tests in `tests/output/enrichStaticSignals.test.ts`.
- **Co-change weight self-normalized to churn** — `computeCriticality` now compares `maxCoChange` to the file's own `churnScore` instead of using absolute thresholds; `+0.15` when the ratio is `≥0.6` with at least 3 co-changes, `+0.10` when the ratio is `≥0.4` with at least 2; makes the heuristic work on any repo regardless of overall activity level (a young repo with tight coupling still scores, a high-churn repo doesn't get inflated by diluted co-changes)
- **Language-aware decision keywords** — `buildBasicNote` now selects decision-keyword tables by `config.language` (English, Portuguese, Spanish, French, German, Italian, with base-language fallback for regional tags like `pt-PT → pt`); conventional-commit prefixes `refactor:`, `revert:`, `perf:` are matched via regex across all languages; teams using non-English commit messages now get their decision-flavoured commits surfaced in `importantDecisions.observed`
- **Expanded side-effect detection** — built-in package list for `hasSideEffects` now covers observability (sentry, datadog, newrelic, opentelemetry, honeycomb, …), analytics (mixpanel, amplitude, segment, posthog, …), message queues (amqp, kafka, nats, sqs, pubsub, …), schedulers (bullmq, bull, agenda, node-cron, …), realtime (socket.io, ws), caches (redis, ioredis, memcached), and feature-flag SDKs (launchdarkly, growthbook, unleash, optimizely); previous hardcoded list of 5 SaaS names replaced
- **Configurable analysis hints** — new `analysis` block in `ai-notes.config.ts` with `sideEffectPackages` and `apiCallPatterns` (regex fragments) so teams can teach braito about their internal SDKs without forking; user entries merge with the built-in defaults; validated by the Zod schema; propagated through `generate` and `watch` into TS/Python/Go analyzers
- **`"refactoring"` keyword in decision detection** — `DECISION_COMMIT_KEYWORDS` now includes `"refactoring"`; commit messages like `"refactoring persistence layer to accept external IDs"` now correctly surface as `importantDecisions` evidence
- **Consolidated co-change evidence in `knownPitfalls`** — per-file `"Co-changed Nx with X"` entries removed from `knownPitfalls.evidence`; a single summary line remains (`"Co-changed with N files (see impactValidation)"`) and the detailed per-file list stays in `impactValidation.evidence` only — eliminates duplicated evidence across both fields
- **Static-note enrichment for side effects and API calls** — `buildBasicNote` now adds contextual purpose hints (`"Has side effects (module-level execution)"`, `"Makes API calls: …"`) when `purposeObserved` is otherwise sparse; improves signal density of static-only notes (below `llmThreshold`) without requiring LLM synthesis
- **LLM cost/duration logging** — `generate` now aggregates and logs LLM usage at the end: total cost (USD), total LLM time (seconds), and number of calls; backed by `LLMUsage` type with `durationMs` and `costUsd` fields, and `onUsage` callback in `synthesizeFileNote`
- **Relative paths in notes** — `buildBasicNote` accepts an optional `root` parameter and converts all absolute paths (reverse deps, direct deps, co-changed files, related tests) to relative paths for portable, readable notes
- **Index uses inferred purpose** — `buildIndex` now prefers `purpose.inferred[0]` over `purpose.observed[0]` for the purpose summary column, showing richer LLM-derived descriptions when available
- **Dedup observed/inferred in merge** — `synthesizeFileNote` merge strategy now filters LLM `inferred` items that duplicate `observed` items (case-insensitive), and deduplicates `evidence` entries by detail text
- **Risky commits kept as evidence only** — `buildBasicNote` no longer promotes risky commit messages (hotfix, revert, etc.) to `knownPitfalls.observed`; they remain in `knownPitfalls.evidence` for LLM reasoning without cluttering the human-facing note
- **Stronger language directive** — system prompt now places the language instruction at both the top and bottom, with stronger wording ("IMPORTANT: You MUST write ALL text content strictly in {language}") to override ambient provider preferences (e.g. Claude CLI user memory)

- **Aggregators now derive from disk (single source of truth)** — refactored `runGenerate` so `index.json`, `search-index.json`, and `graph.json` node metadata are built from `loadAllNotesFromDisk(root, outputDir)` after the synthesis loop, instead of from an in-memory `notes[]` populated only by files the loop touched. Eliminates the entire class of bugs where `--filter` / `--force` / cache-only runs accidentally shrank the repo-wide artifacts because their loops processed only a subset. Removed three workaround code paths added in earlier fixes (cache-hit disk-reload inside the loop, post-loop unfiltered reload, and `cached?: boolean` plumbing in the result tuple); the disk read is one `fs.readdir` walk + parallel `JSON.parse` (~50ms for 74 files in the example NestJS repo). New helper at `src/core/output/loadAllNotesFromDisk.ts` skips reserved filenames (`index/graph/search-index/divergences.json`) and returns parse errors per-file instead of aborting; the three regression tests for cache-only / `--filter` / `--force --filter` now act as behavioral contracts on the new design.

- **Confidence calibration** — reweighted `computeCriticality` in `buildBasicNote.ts`: hooks raised from `+0.15` to `+0.2`; untested files with consumers now penalized `+0.15` instead of flat `+0.1`; `apiCalls.length > 0` now adds `+0.1` (was not factored in despite being extracted by AST); untested files with no consumers reduced to `+0.05` to avoid over-inflating leaf files

### Fixed
- **`get_architecture_context` returns empty fields in `topCriticalFiles`** — handler resolved per-file notes from `entry.filePath` (an absolute path), so `path.resolve(root, outputDir, '/abs/.../foo.ts.json')` collapsed to the source-file path, every `fs.readFileSync` threw `ENOENT`, and the silent catch returned `{}` — leaving `purpose`/`invariants`/`knownPitfalls`/`sensitiveDependencies`/`importantDecisions` empty even when the underlying notes were richly populated. Now uses `entry.relativePath` (matching `search`/`get_impact`), adds the same path-traversal guard those handlers have, and merges `observed + inferred` so LLM-synthesized fields actually surface (the previous code only ever showed the static `observed` slice — for top-criticality files this meant showing `"Export: class ProductService"` instead of the rich purpose synthesized by claude-opus-4-6); response now includes `relativePath` for consistency with the other tools; regression test `get_architecture_context populates topCriticalFiles fields from per-file notes` in `tests/e2e/cli.test.ts`.
- **`--force` wipes cache for files outside `--filter`** — `runGenerate` initialised `noteHashStore` and `analysisStore` as empty Maps when `args.force` was set; combined with `--filter` this was catastrophic — only the filtered file's hash got re-added before `saveCache`, leaving every other entry gone, so the next plain run had to re-synthesize the entire repo via the LLM (verified: `--force --filter` against the example NestJS repo cleared 73/74 cache entries). Both stores now load from disk unconditionally; `--force` is enforced inside the loop via `!args.force && noteHashStore.get(...) === currentHash`, so `--force` still re-runs everything when used alone, but `--force --filter` only touches matched entries; regression test `--force --filter does not wipe cache entries for unfiltered files` in `tests/e2e/cli.test.ts`.
- **`--filter` rerun shrinks index/search-index/graph to filtered set** — same root cause as the cache-only bug: `notes[]` only collected files seen by the loop, so a `generate --filter src/foo` rerun rebuilt the index with just the matched files and dropped every other note's metadata. After the loop, `runGenerate` now reloads on-disk notes for unfiltered analyses and merges them into `notes[]` before `buildIndex` / `buildSearchIndex` / `writeGraph` run; regression test `filter rerun preserves existing notes for unfiltered files in index` in `tests/e2e/cli.test.ts`.
- **`claude-cli` synthesis silently fails when model prepends prose** — `synthesizeFileNote.parseJSON` only stripped markdown fences, so responses like `"Looking at this file...\n\n{...}"` or `"Now let me analyze...\n\n\`\`\`json\n{...}\n\`\`\`"` triggered `JSON Parse error: Unexpected identifier "Looking"` / `"Now"` and silently fell back to the static note — losing LLM enrichment for the most critical files in the run. Extracted parser into `src/core/llm/parseJsonResponse.ts` that walks every `{` candidate with a string-and-escape-aware brace counter and tries `JSON.parse` until one succeeds; system prompt strengthened with `"The first character of your response MUST be \`{\` and the last character MUST be \`}\`"` plus an explicit "do not begin with phrases like 'Looking at this file', 'Now let me'..."; 10 tests in `tests/llm/parseJsonResponse.test.ts` cover prose prefix, prose suffix, fenced+prose, nested braces inside string values, and escaped quotes. Verified end-to-end on the example NestJS repo: 3 previously-failing files (top criticality `products.service.ts` 0.95, `product.entity.ts` 0.86, `product-data-request.dto.ts` 0.73) now synthesize cleanly via `claude-opus-4-6`.
- **Cache-only `generate` run wipes index/search-index/graph metadata** — when every source file matched the cache, `notes[]` was empty so `buildIndex` and `buildSearchIndex` overwrote `.ai-notes/index.json` and `.ai-notes/search-index.json` with empty payloads, and `graph.json` lost its `nodeMetaMap` (`domain`/`criticalityScore`). Cache-hit branch in `runGenerate` now reloads the existing per-file note JSON from disk, pushes it into `notes[]`, and counts it as `skipped` (not `written`); regression test `cache-only re-run preserves index.json and search-index.json` in `tests/e2e/cli.test.ts`. Affected MCP (`get_index`, `search`, `get_impact`) and the UI graph tab on every cache-only re-run.
- **`withDefaults` drops `llm` config** — `withDefaults` in `src/core/config/defaults.ts` was reconstructing the config object without `llm` and `maxSourceLines` fields, silently dropping LLM configuration and causing all runs to use static-only mode; both fields now passed through
- **LLM evidence schema too strict** — `evidenceItemSchema.type` now uses `.catch('code')` so unknown `type` values returned by the LLM (e.g. `'import'`, `'external'`) are coerced to `'code'` instead of failing Zod validation and silently falling back to the static note; 3 tests in `tests/llm/aiNoteSchema.test.ts`
- **Missing `signatures` field in Python/Go analyzers** — added `signatures` to both analyzer return objects to satisfy `StaticFileAnalysis` type contract; LLM prompts no longer show "none extracted" for Python/Go files
- **Absolute paths in Impact Validation** — `getCoChangedFiles` was calling `path.resolve(root, p)` on paths already returned as relative by `git diff-tree`, producing absolute paths in `impactValidation.observed` and `knownPitfalls.observed`; fixed by keeping the git-relative path as-is

- **Go local import detection** — replaced the incorrect `includes('./')` heuristic with `go.mod`-based module path matching; `getGoModuleName` walks up the directory tree to find the nearest `go.mod` and classifies imports starting with the module prefix as local; falls back to `./`/`../` heuristics when no `go.mod` is found
- **`parseFile` error resilience** — wrapped ts-morph parsing in try/catch in `parseFile`; on any error a warning is logged via `logger.warn` and an empty `StaticFileAnalysis` is returned instead of crashing the pipeline
- **Alias resolution in watch mode** — `watch.ts` now calls `loadBundlerAliases(root)` and passes aliases to `buildDependencyGraph`, matching the behavior of `generate.ts`; bundler aliases (Vite, Webpack, Metro) are now resolved correctly in watch mode
- **Incremental graph rebuild in watch mode** — `watch.ts` now calls `updateDependencyGraph` (new export) on each file change instead of rebuilding the full graph; only the changed file's dependency entry is updated, then the reverse graph is rebuilt from the patched dep graph
- **Graph test using real temp files** — `buildDependencyGraph.test.ts` now creates actual temp files so `resolveImportPath` can resolve them via `fs.existsSync`; added coverage for `updateDependencyGraph`

### Removed
- **`get_overview` MCP tool** — removed orphan tool that attempted to read `overview.json` which was never generated by the pipeline; `get_architecture_context` provides equivalent and richer functionality
